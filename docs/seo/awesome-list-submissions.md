## Submission Checklist

### Pre-Submission

- [ ] Verify each awesome list repo accepts PRs (check CONTRIBUTING.md)
- [ ] Fork the target awesome list repo
- [ ] Read the list's submission guidelines
- [ ] Find the correct alphabetical position in the list
- [ ] Ensure the entry format matches the list's convention

### Entry Format (generic)

```markdown
- [pepwiki/wikisites](https://github.com/pepwiki/wikisites) - Peptide molecular weight calculator with TypeScript and Python implementations. Zero dependencies, monoisotopic residue masses, molecular formula generation. [MIT License]
```

### Per-List Steps

For each target list:

1. **Fork** the awesome list repo
2. **Clone** the fork locally
3. **Create a branch** for the addition
4. **Add the entry** in the correct section, alphabetically
5. **Commit** with a descriptive message
6. **Push** and open a PR
7. **Fill out** the PR template completely
8. **Respond** to reviewer feedback promptly

### Post-Submission

- [ ] Monitor PR status for each submission
- [ ] Respond to any review comments within 24 hours
- [ ] Make requested changes if needed
- [ ] Add merged PR links to project README
- [ ] Track which lists accepted/denied for future strategy
