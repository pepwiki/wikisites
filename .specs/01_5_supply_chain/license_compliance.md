## 3. Attribution Files

### 3.1 NOTICE File Generation

A `NOTICE` file should be generated containing all copyright notices and license texts. Use `license-checker` or similar tool:

```bash
# Generate NOTICE file
npx license-checker --production --json > licenses.json
npx license-checker --production --customPath '{"licenseText": ""}' --csv > licenses.csv
```

### 3.2 Required Inclusions

For each dependency, the following must be included in the distributed NOTICE file:

```
[Package Name]
Copyright (c) [Year] [Copyright Holder]
Licensed under the [License Name]

Full license text:
[License Text]
```

## 7. License Violation Response Plan

### 7.1 Detection

- CI pipeline runs `license-checker` on every build
- Renovate/Dependabot PRs include license check
- Manual review of new dependencies

### 7.2 Response

1. **Identify:** Determine which package has the problematic license
2. **Assess:** Determine if the license is compatible with MIT
3. **Mitigate:**
   - If compatible: Document and continue
   - If incompatible: Find alternative package or restructure code
   - If copyleft: Remove dependency and find permissive alternative
4. **Report:** Document incident and resolution

### 7.3 Escalation

- If a newly introduced dependency has a copyleft license, block the PR
- If a critical dependency changes license, evaluate alternatives within 30 days
- If a license violation is discovered post-release, prepare patch release
