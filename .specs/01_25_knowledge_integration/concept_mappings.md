# Cross-Lingual Knowledge Integration: Concept Mappings

**Document ID:** KI-01-25-CM  
**Version:** 1.0.0  
**Date:** 2026-06-19  
**Status:** COMPLETE  
**Classification:** Internal — Phase 1.25 Concept Standardization

---

## Overview

Standardized concept mapping for all 45 terms across 5 domains. Each mapping includes the EN canonical term, translations for ZH/JA/AR, confidence scores, source references, and ambiguity flags.

**Confidence Score Legend:**
- **0.95-1.00**: Official documentation, verified across multiple sources
- **0.85-0.94**: Verified against primary references
- **0.75-0.84**: Community standard, widely accepted
- **0.60-0.74**: Generated from linguistic rules, requires review
- **<0.60**: Uncertain, requires expert validation

## 6. Ambiguity Summary

| Concept | Ambiguity Type | Resolution |
|---------|---------------|------------|
| Highlight | Polysemy (search vs annotation) | Same visual metaphor; same term in all languages |
| Settings | Polysemy (user/theme/plugin) | Context-dependent; same term in all languages |
| GDPR | Conceptual mismatch (JA) | JA 個人情報保護法 ≈ GDPR; functionally equivalent |
| Design Token | Term variation (ZH) | "设计令牌" preferred over "设计代币" |
| Schema | Polysemy (ZH) | "架构" = schema/architecture; context-dependent |
